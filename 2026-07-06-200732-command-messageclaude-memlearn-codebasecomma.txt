  Key Design Decisions

  - Uses uv for dependency management
  - 1bit-trainer/ is a Python package, importable by external scripts
  - BitNet b1.58 uses ternary weights (-1, 0, +1) stored in INT2 packed format
  - SubQSA (Sub Quadratic Sparse Attention) builds content-dependent dynamic
  sparse patterns via locality-sensitive hashing
  - Long-context support via RingAttention with sequence parallelism
  - Steam "massively parallel" hybrid: Intake/Process/Decode pipeline with
  distributed sequence processing
  - Modular trainer hierarchy: base → specialized trainers (pretraining, SFT,
  DPO, GRPO, KD)
  - GPU optimization via custom Triton fused kernels for BitLinear (INT8
  forward, FP8 backward) and sparse attention
  - Knowledge graph persistence via graphify for codebase analysis and
  architecture tracking

  Testing

  - Pure Python functions: unit tests (fast, no GPU required)
  - Triton kernels: correctness tests (GPU required, skipped if unavailable)
  - Full integration: GPU integration tests (GPU required)
  - Test files: tests/test_*.py for each module

  claude-mem

  claude-mem adds persistent cross-session memory to Claude Code via a local
  SQLite database + vector search plugin. It learns from observations, answers
  queries about past sessions, and auto-injects context into new sessions so you
  don't have to repeat yourself.

  How it works

  1. Observations (automatic learning)

  The plugin records observations automatically. Every PostToolUse hook fires an
  observation-generation job, and every 90 seconds a background worker extracts
  structured knowledge from that session's transcript. Observations are tagged
  by project, kind (decision, bugfix, feature, refactor, discovery, change), and
  concept — they feed the memory database used by search and injection.

  2. Auto-inject (context priming)

  Each new session starts with a 25-token memory header in CLAUDE.md. On your
  second session in a project, claude-mem searches for relevant past
  observations based on what you're working on now and injects the top matches
  into your context — so important context carries forward without you asking
  for it. Before auto-inject is ready, you can run /mem-search to query
  manually.

  3. Memory search (querying)

  The /mem-search slash command lets you search past observations by project,
  type, concept, or date range. It uses FTS (SQLite full-text search) against
  the local database — no external servers, no API calls, everything stays on
  your machine. Use it to recall past decisions, find how you solved a problem
  before, or check what's been done.

  4. Knowledge corpora (priming)

  Use /build-corpus to compile focused "brains" from observations — filter by
  project, type, concept, or semantic query. Then /prime-corpus loads a corpus
  into an AI session, and /query-corpus lets you ask questions against it. Great
  for deep-diving into specific topics or preparing context for a complex task.

  5. Vector search (plugin)

  The MCP server exposes search (fast index scan), timeline (context around a
  result), and get_observations (full detail fetch) — a 3-layer workflow that
  keeps token usage low. Use these programmatically when you need structured
  memory access from agents or scripts.

     Ultimate AI Model Trainer — Bug Fixes, Optimizations & Test Gaps

     Date: 2026-07-06
     Context: Comprehensive codebase audit across all 28 source files. Three
     parallel agents analyzed bugs/errors (30 findings), optimization
     opportunities (28 findings), and test coverage gaps (~4% coverage, 30+
     untested paths). This plan prioritizes critical/high-severity items that
     must be fixed ASAP, followed by important optimizations and test coverage
     improvements.

     ---
     PHASE 1: CRITICAL BUG FIXES (Fix Immediately)

     1.1 Fused Triton Kernel Severs Autograd Graph — Model Untrainable on GPU

     - File: ultimate_trainer/bitlinear.py:106-107
     - Bug: fused_bitlinear_forward() writes output into torch.empty() tensor
     with no grad_fn. The backward graph is severed — ALL BitLinear layers
     receive zero gradients. This means the entire ultimate_trainer pipeline
     cannot learn on GPU with the fused kernel.
     - Fix: Guard fused path with not self.training. Only use it during
     eval/inference:
     # Current (broken during training):
     if self.quantize_activations and x.is_cuda and _HAS_FUSED_KERNEL:
         return fused_bitlinear_forward(x, self.weight, self._gamma, self.bias)
     # Fixed:
     if not self.training and x.is_cuda and _HAS_FUSED_KERNEL:
         return fused_bitlinear_forward(x, self.weight, self._gamma, self.bias)
     - Impact: Without this fix, ultimate_trainer GPU training is impossible.
     Loss will never decrease.

     1.2 Zero-Input NaN Bug in Activation Quantization

     - Files: ultimate_trainer/bitlinear.py:28-34, 1bit-trainer/model.py:62-85
     - Bug: When activations are all zeros: abs_max = 0 → scale = 0/(q_max+1e-5)
     ≈ 0 → act/scale = 0/NaN → NaN. Propagates through STE into output, then
     into loss and gradients. Common at initialization and with ReLU dead
     neurons.
     - Fix: Add epsilon to the denominator properly:
     scale = abs_max / (q_max + 1e-5)
     # Should be:
     scale = abs_max.clamp(min=1e-5) / q_max
     - Or equivalently: scale = abs_max / q_max; scale = 
     scale.clamp(min=1e-5/q_max)
     - Impact: Training can collapse to NaN from a single batch with zero
     activations.

     1.3 Checkpoint Resume Trains Max Steps Instead of Remainder

     - File: train_longctx.py:283 vs 265
     - Bug: After resume, training loop uses for step in 
     range(self.tc.max_steps) starting from 0, but global_step is already
     restored. Model trains max_steps EXTRA instead of max_steps - global_step.
     Scheduler advances past correct position.
     - Fix:
     remaining = self.tc.max_steps - self.global_step
     for step in range(remaining):
         ...
     - Also log global_step instead of local step.

     1.4 Dataset Exhaustion Causes Unhandled StopIteration Crash

     - File: train_longctx.py:286
     - Bug: batch = next(data_iter) raises StopIteration when
     FineWebLongCtxDataset exhausts its max_docs before max_steps complete. No
     try/except.
     - Fix: Add iterator recreation:
     try:
         batch = next(data_iter)
     except StopIteration:
         data_iter = iter(self.dataset)
         batch = next(data_iter)

     1.5 SubQSA Gate Normalization Missing Epsilon — NaN Risk

     - File: subqsa_trainer/subqsa.py:158
     - Bug: g / g.sum(dim=-1, keepdim=True) has NO epsilon. If all gate scores
     approach zero, division by zero → NaN.
     - Fix: g = g / (g.sum(dim=-1, keepdim=True) + 1e-8) (matches
     ultimate_trainer/subqsa.py)

     1.6 NSA Fused Kernel Is Dead Code — Never Calls parallel_nsa

     - File: ultimate_trainer/subqsa.py:40-45, 272-277
     - Bug: _HAS_NSA_KERNEL imports fla.ops.parallel_nsa but NEVER calls it. The
     "fused" path calls _nsa_fused_forward() which is the same pure PyTorch

     PHASE 2: HIGH-PRIORITY OPTIMIZATIONS

     2.1 Synchronize Ternary Weight Refresh Across All BitLinear Modules

     - File: ultimate_trainer/bitlinear.py:100-104
     - Issue: Each BitLinear tracks its own _quant_step, causing 210+
     independent gamma recomputations per refresh cycle. Each reads full weight
     matrix from HBM.
     - Optimization: Pass a global training step counter and batch all gamma
     computations together. Increase quant_update_freq to 100 for large models.
     Save 3-5% step time.

     2.2 Fuse Gamma Computation + Ternary Quantization into Single Kernel

     - File: kernels/ternary_matmul.py:189-200,
     ultimate_trainer/bitlinear.py:83-90
     - Issue: compute_gamma reads full weight matrix. _refresh_ternary_weights
     reads it again. Two sequential HBM reads of same data.
     - Optimization: Extend gamma Triton kernel to also produce ternary weights
     in one pass. Eliminate one full weight matrix HBM read per refresh.

     2.3 Replace SelectionBranch Manual Einsum with FlashAttention

     - File: ultimate_trainer/subqsa.py:133-138
     - Issue: Separate einsum("bhtd,bhtld->bhtl") + softmax +
     einsum("bhtl,bhtld->bhtd") — three kernel launches, materializes full
     attention score matrix (336MB per layer at 4K).
     - Optimization: Use single F.scaled_dot_product_attention call. Reshape
     k_sel to (B, H, T, topk*lp, D). One fused FlashAttention kernel, scores
     never materialized. 2-3× branch speedup.

     2.4 Cache Sliding Window Mask

     - File: ultimate_trainer/subqsa.py:158-165
     - Issue: (T, w) mask allocated every call via torch.where. 2M elements at
     4K, 30 layers = 60M per forward pass. Custom masks force SDPA off
     FlashAttention fast path.
     - Optimization: Pre-allocate mask buffer keyed by (T, w). Better:
     restructure to avoid mask entirely by using FlashAttention native causal
     masking on properly sliced tensors.

     2.5 Cache RoPE Cos/Sin Tables

     - File: 1bit-trainer/model.py:186-200 (shared via importlib)
     - Issue: cos() and sin() computed from scratch every forward call. ~1ms per
     RoPE × 2 (Q + K) × 30 layers = 60ms per forward.
     - Optimization: Precompute full cos/sin table for max_seq_len in __init__,
     index with position_ids. Replace GPU trig with cheap gather. Saves
     ~60ms/forward at 4K.

     2.6 Perform Compression Attention at KV-Head Resolution

     - File: ultimate_trainer/subqsa.py:256-259
     - Issue: K_cmp expanded from 5 KV heads to 20 heads BEFORE attention. 4×
     memory expansion then attention produces (B, 20, T, n_cmp) score matrix =
     20M elements.
     - Optimization: Do compression attention at 5 KV-head resolution, expand
     result to 20 heads after. Saves ~7GB for 30-layer model.

     2.7 Implement Activation Checkpointing

     - Files: ultimate_trainer/train.py, train_longctx.py
     - Issue: All intermediate activations materialized simultaneously. For 2B
     model at 4K: ~12GB activation memory. For 1M context: prohibitively large.
     - Optimization: Use torch.utils.checkpoint on transformer layers.
     Checkpoint every 4 layers → 75% activation memory reduction with ~15%
     recomputation overhead. Essential for 1M-context training.

     2.8 Make _w_ternary Non-Persistent Buffer

     - Files: 1bit-trainer/model.py:126, ultimate_trainer/bitlinear.py:67
     - Issue: _w_ternary duplicates weight matrix in state_dict, doubling
     checkpoint size. For 2B model: 8GB→16GB checkpoints.
     - Optimization: Set persistent=False for _w_ternary buffer. Recompute on
     load from master weight.

     2.9 Reduce CompressionBranch MLP Input Dimension

     - File: ultimate_trainer/subqsa.py:71-72
     - Issue: nn.Linear(head_dim * block_len, head_dim * 2) where 128*32=4096
     input features. 5.3 GFLOPs per layer for φ_k alone. At 30 layers: 320
     GFLOPs for compression MLP (K+V).
     - Optimization: Reduce block_len from 32 to 16, halving input to 2048 and
     cutting FLOPs by ~2×. Or use bottleneck: pool first, then project.
